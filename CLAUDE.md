# 게임 개발 기획서 — 똥피하기 (Dodge the Poop)

## 1. 개요

| 항목 | 내용 |
| --- | --- |
| 게임명 | 똥피하기 (Dodge the Poop) |
| 장르 | 캐주얼 아케이드 / 회피(Dodge) |
| 플랫폼 | 웹 브라우저 (PC + 모바일) |
| 목표 유저 | 라이트 유저, 짧은 시간에 즐기는 캐주얼 게이머 |
| 한 판 플레이 시간 | 30초 ~ 3분 (생존 시간에 비례) |
| 핵심 재미 | 위에서 떨어지는 똥을 좌우로 피하며 생존 시간을 늘리는 반사신경 게임 |

## 2. 게임 컨셉

화면 상단에서 똥(💩) 오브젝트가 계속 떨어지고, 플레이어는 하단의 캐릭터를 좌우로 움직여 이를 피한다. 시간이 지날수록 낙하 속도와 생성 빈도가 증가하며 난이도가 올라간다. 똥에 맞으면 목숨(하트)을 잃고, 목숨을 모두 소진하면 게임 오버. 생존 시간과 피한 개수로 점수를 매긴다.

## 3. 핵심 게임플레이

### 3.1 조작
- **PC**: 방향키(←/→) 또는 A/D 키로 캐릭터 좌우 이동
- **모바일**: 화면 터치/드래그, 또는 좌우 절반 탭으로 이동
- 캐릭터는 화면 하단 고정 Y좌표에서 X축으로만 이동

### 3.2 낙하 오브젝트
- 기본 똥(💩): 일반 낙하 속도, 맞으면 목숨 -1
- 황금 똥(✨💩, 확률 낮음): 맞으면 보너스 점수 + 획득 시 무적 아이템처럼 처리 가능(선택 사항)
- 낙하 위치는 랜덤 X좌표에서 생성, 화면 하단 도달 시 제거(오프스크린 처리)

### 3.3 난이도 곡선
| 경과 시간 | 낙하 속도 배율 | 생성 간격 |
| --- | --- | --- |
| 0~15초 | 1.0x | 1.2초 |
| 15~30초 | 1.3x | 0.9초 |
| 30~60초 | 1.6x | 0.6초 |
| 60초~ | 매 30초마다 +0.2x | 매 30초마다 -0.1초 (최소 0.2초) |

### 3.4 목숨 & 점수
- 시작 목숨: 3개 (하트 아이콘 UI로 표시)
- 똥에 맞으면 목숨 -1, 잠깐 무적 시간(1초) 부여 후 재충돌 방지
- 점수: 생존 시간(초당 +10점) + 회피한 똥 개수(개당 +1점)
- 목숨 0이 되면 게임 오버, 최종 점수 표시

## 4. 화면 구성 (와이어프레임 레벨)

1. **타이틀 화면**: 게임 제목, "시작하기" 버튼, 최고 점수 표시
2. **게임 화면**: 상단에 점수/목숨 HUD, 중앙에 플레이 영역, 캐릭터 및 낙하 오브젝트
3. **게임 오버 화면**: 최종 점수, 최고 점수 갱신 여부, "다시하기" / "타이틀로" 버튼

## 5. 기술 스택 (제안)

- **렌더링**: HTML5 Canvas (2D Context) 또는 간단한 DOM+CSS 애니메이션
- **언어**: Vanilla JavaScript (프레임워크 없이 시작, 추후 필요 시 확장)
- **구조**: `index.html` + `style.css` + `game.js` 단일 페이지 구성
- **게임 루프**: `requestAnimationFrame` 기반
- **충돌 판정**: 사각형(AABB) 충돌 검사
- **저장**: `localStorage`로 최고 점수 저장

## 6. 파일 구조 (예정)

```
/
├── index.html      # 게임 진입점, 캔버스 및 UI 마크업
├── style.css        # 레이아웃, HUD, 화면 전환 스타일
└── game.js           # 게임 루프, 입력 처리, 충돌 판정, 상태 관리
```

## 7. MVP 범위

1. 캐릭터 좌우 이동 (키보드)
2. 똥 오브젝트 낙하 및 충돌 판정
3. 목숨 시스템 및 게임 오버 처리
4. 점수 계산 및 화면 표시
5. 난이도 시간 경과에 따른 증가
6. 타이틀 / 게임 / 게임오버 화면 전환

## 8. 향후 확장 아이디어 (Post-MVP)

- 모바일 터치 컨트롤 최적화
- 사운드 이펙트 및 배경음악
- 콤보 시스템 (연속 회피 시 점수 배율 증가)
- 아이템: 방패(일시 무적), 자석(똥을 옆으로 밀어냄) 등
- 캐릭터 스킨 선택
- 온라인 랭킹(백엔드 필요)
